# 🚇 深圳地铁实时客流模拟系统

一个基于原生JavaScript、Canvas和Web数据可视化技术构建的前端练手项目。

## ✨ 项目简介
这是一个模拟展示深圳地铁实时客流的单页面应用(SPA)。通过可视化手段，直观呈现了深圳主要地铁线路的站点、换乘信息及模拟的实时客流数据。

**核心价值**：本项目专注于**原生技术栈**的深入实践，旨在夯实前端基础，避免依赖框架黑箱。

## 🔧 技术栈
- **前端三件套**: HTML5, CSS3 (Flex/Grid布局), JavaScript (ES6+)
- **核心特性**: 模块化开发 (ES Modules)、Canvas 2D 绘图、异步编程 (Async/Await)
- **数据交互**: Fetch API、动态JSON数据加载
- **工具与规范**: Git版本控制、语义化提交

## 🗺️ 核心功能
1.  **线路可视化**：展示深圳地铁1、2、3、5、11号线，按官方配色渲染。
2.  **交互式查询**：
    - 点击线路按钮，动态切换并高亮显示对应线路的所有站点。
    - 集成实时搜索框，支持模糊查询站点，并带有**防抖(debounce)**优化。
3.  **动态效果**：
    - 页面实时时钟，每秒更新。
    - 模拟的客流热力图展示。

## 🤔 我通过这个项目学到了什么？
（**这是最能打动人的部分！请用你的话写下真实体会**）
- **工程化思维**：理解了模块化(ES Modules)如何将复杂功能拆解为 'dataLoader.js', 'lineList.js', 'stationList.js' 等独立、可维护的模块。
- **核心交互原理**：掌握了事件监听、回调函数、状态管理('currentSelectedLine')如何协同工作，驱动单页面应用(SPA)的视图更新。
- **性能优化实践**：在搜索功能中亲手实现了**防抖(debounce)**，理解了其避免频繁函数调用、提升性能的原理。
- **异步编程**：深入使用 'async/await' 处理数据加载，让异步代码拥有同步代码的可读性。
- **DOM操作**：熟练运用 'document.createElement', 'appendChild', 'innerHTML' 等API进行动态渲染。
